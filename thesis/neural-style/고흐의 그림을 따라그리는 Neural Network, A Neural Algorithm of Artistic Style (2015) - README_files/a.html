                                                질문이 좀 많이 헷갈리네요. input image가 계속 update되냐는 질문으로 이해하고 답변해드리도록하겠습니다.
                                                이 논문에서 가장 헷갈리기 쉬운 부분이, optimization을 통해 구하고 싶은 parameter가 image이고, 나머지는 전부 고정되어있다는 것입니다.
                                                optimization function을 보시면, Lc는 x와 p의 차이, Ls는 x와 a의 차이를 나타냅니다. (x: optimization을 통해 얻고 싶은 값, a: 고흐의 별이 빛나는 밤, p: 튀빙겐 사진) 그냥 퉁쳐서 전체 loss를 L(x,a,p) 라고 하면, x는 아래와 같은 optimization problem의 해가 됩니다.
                                                x = argmin_x' L(x',a,p)
                                                이때, 이 optimization을 구하기 위해 gradient descent를 사용하게 되는거죠. 그렇기 때문에 dL/dx 를 계산해야하는겁니다.
                                                dL/dx 는 dLc/dx와 dLs/dx로 구성되어 있으니, 각각에 대하여 계산을 해야하는데, 그냥 계산하면 어려우니까 본문에 있는대로 chain rule을 써서 backpropagation을 사용합니다. 이때, 해당 layer에 해당하는 미분값은 feature map에 대한 어떤 관계식으로 계산할 수 있게 되는거구요.
                                                일반 neural network에서 weight를 parameter로 하여 계속 forward backward를 반복하면서 gradient descent를 진행하듯, 이 work도 image를 parameter로 하여 계속 forward backward를 반복하게 됩니다. 이때, 초기값을 그냥 random image로 주었을 뿐이구요. 따라서 질문해주신것처럼 iteration이 진행되면서 계속 x의 gradient를 계산하게 되니까, x = x + gamma * dL/dx로 update를 하게 됩니다. (gamma: learning rate)
                                                질문에 충분한 답이 되셨으면 좋겠습니다

                                                    CONV filter weight 는 전혀 건드리지 않는다고 확실히 말해주시니 이제 좀 이해가 될거같기도 합니다. 제가 이해한걸 적어보겠습니다.
                                                    *Conv layer n 에서 일어나는 연산을 함수로 표현하여 CONn() 으로 두겠습니다.
                                                    *비교할 Content img 의 CONV4 layer 에서의 feature map
                                                    P4 = CON4(CON3(CON2(CON1(p))))
                                                    여기서 P4 는 한번만 계산하고나면 사실상 상수가되겠네요,

                                                    * optimize 할 input img 의 CONV4 layer 에서의 feature map
                                                    F4 = CON4(CON3(CON2(CON1(x))))

                                                    * Lc = (F4-P4)^2
                                                    = ( CON4(CON3(CON2(CON1(x)))) - P4)^2
                                                    xnew = x + gamma*dLc/dx
                                                    = x + gamma* 2( CON4(CON3(CON2(CON1(x)))) - P4) * CON4'(CON3(CON2(CON1(x)))) * CON3'(CON2(CON1(x))) * CON2'(CON1(x)) * CON1'(x) 이 되겠네요.

                                                    dLc/dx 를 계산하는 식이 가능한가? 그리고 filter weight 는 어떻게 동시에 optimize 하지? 이 두가지 사이에서 계속 헤메게 된것 같습니다.
                                                    content 가 비슷한 정도를 나타내주는 값인 Lc 는 비교적 쉽게 생각할수있는 Loss function 일거같은데, style 이 비슷한정도를 나타내주는 값인 Ls 는 feature map 끼리의 correlation 이 얼마나 비슷한지를 통해 산출한다는 개념을 어떻게 알아냈는지는 정말 신기한거같습니다.
